### **Rank 2: Defer Selection Geometry Resolution (Medium Priority - Visual Lag Fix)**
* **Impact:** Eliminates the visual "drift" (the 1-frame latency) where selection highlights and handles drag behind the text by exactly one frame.
* **Why:** Since [refreshSelectionForViewportChange](file:///d:/flutter/personal_webpage/packages/tw_primitives/lib/src/selection/selectable_region_fork.dart#L676) runs before the new layout has executed, it queries stale rendering coordinates. Deferring the calculation using a frame callback (e.g., `SchedulerBinding.instance.addPostFrameCallback`) ensures coordinates are queried *after* the viewport has scrolled and updated child layouts.
* **Effort:** Medium. Requires careful integration to ensure state updates (`setState`) are not triggered during the build phase.

### **Rank 3: Throttling Geometry Rebuilds (Low Priority - Performance Optimization)**
* **Impact:** Reduces CPU consumption, minimizes micro-stuttering, and improves FPS on lower-end devices and mobile web browsers during rapid scrolling.
* **Why:** Calling `getTransformTo(null)` on every single scroll update is computationally expensive. Throttling updates (e.g., when the user is not actively dragging) saves valuable rendering time.
* **Effort:** Medium. Requires implementing debouncing/throttling logic during inactive scroll states.
