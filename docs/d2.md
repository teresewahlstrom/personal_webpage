# Terese asks
In my flutter webpage: The bottom row that belongs to the phone UI shows up slightly after my webpage loads and it causes my bottom flowing elements to jump


# chatgpt answers
This is almost certainly **mobile browser viewport resizing**.

On Android/Chrome, the bottom system/browser UI can appear after load, which changes the usable viewport height. If your Flutter web layout uses full-height constraints, bottom-positioned elements jump when the viewport changes.

Use **dynamic viewport units** or avoid hard-locking to initial screen height.

For Flutter web, check these:

```dart
final view = View.of(context);
final bottomInset = view.viewPadding.bottom / view.devicePixelRatio;
```

Then add safe bottom padding:

```dart
Padding(
  padding: EdgeInsets.only(bottom: bottomInset + 12),
  child: YourBottomInputBar(),
)
```

If your root layout uses something like:

```dart
height: MediaQuery.of(context).size.height
```

or a `Stack` with `Positioned(bottom: 0)`, that is likely the cause.

Better pattern:

```dart
Scaffold(
  resizeToAvoidBottomInset: true,
  body: SafeArea(
    bottom: true,
    child: Column(
      children: [
        Expanded(child: Content()),
        BottomInputBar(),
      ],
    ),
  ),
)
```

For Flutter web specifically, also consider setting the page CSS height to dynamic viewport height:

```css
html, body {
  height: 100dvh;
  margin: 0;
  overflow: hidden;
}
```

But the more robust fix is: **do not anchor critical UI directly to raw viewport bottom**. Let `SafeArea` / `viewPadding.bottom` reserve space for browser/system UI.

Likely diagnosis: your bottom chat/input bar is positioned against the initial viewport, then Android reveals/adjusts the navigation/browser bottom area, reducing available height and forcing a layout recalculation.
