# Viewport, Zoom, and Chat Follow-Up

## Current Remaining Themes

- Mobile/browser viewport behavior:
  - keyboard jumps when opening the app or scrolling,
  - temporary white block during tab switching,
  - Android Chrome desktop-mode sizing looks too zoomed out.
- Browser zoom behavior:
  - Android Chrome pinch zoom is not available,
  - Desktop Chrome `Ctrl` + wheel zoom is not available.
- Repo/package cleanup:
  - unify color, line, shadow, animation, glow, and font token usage,
  - consider whether markup decoration constants belong in `tw_primitives`.

## tw_chat Items Likely Solved

- right-clicking selected chat text selecting main-page text,
- copy target confusion between chat and page selection,
- chat text selection dropping too easily,
- truncated bubbles being scrollable,
- hidden truncated text creating selection handles below visible content,
- truncated bubble copy still needing the full message body.

These should stay on the manual regression list until they are proven in real deployment.

## Architecture Direction

Keep `TwScrollArea` focused on scroll ownership:

- custom scrollbar,
- optional `SingleChildScrollView`,
- scrollbar overlay/track,
- scroll physics and scrollbar styling.

Do not put page-level browser/keyboard white-flash behavior directly into `TwScrollArea`. That behavior is more about the whole app surface than one scrollable region.

Likely split:

```text
PageScaffold
  owns app/page-level interaction policy:
  - browser/page zoom policy
  - page scroll focus
  - page selection clearing
  - chat interaction handoff
  - background exposed if Flutter drops a frame

TwScrollArea
  owns reusable scroll chrome and scroll behavior

TwPanZoomArea / TwInteractiveScrollArea
  optional future local pan/zoom primitive for canvas-like content
```

## Next Useful Spike

Start small around `web/index.html` and `PageScaffold` before creating a new primitive:

- Does adding a normal viewport meta tag restore pinch zoom?
- Does changing `html/body/#app` `overflow: hidden` or `position: fixed` affect zoom or the white flash?
- Does Flutter page scrolling still behave if browser zoom is allowed?
- Does desktop `Ctrl` + wheel fail because Flutter intercepts it, or because the browser/page setup prevents it?

If this spike works, codify the app-level decision in `PageScaffold` and keep any DOM-specific code in a small web helper.
