Looking at the current implementation, the footer/header/keywords-frame outline lines are set by these color tokens in app_color_theme.dart:

| Element | Token | Current Light Value | Current Dark Value |
|---------|-------|---------------------|-------------------|
| **Header border** | `lineSubtleTertiary` | `Color(0x40394183)` | `Color(0x397199FF)` |
| **Footer border** | `lineSubtleSecondary` | `Color(0x40394183)` | `Color(0x397199FF)` |
| **Keywords frame border** | `lineSubtleSecondary` | `Color(0x40394183)` | `Color(0x397199FF)` |

These are applied via:
- _page_header.dart: `AppLineTheme.subtleTertiaryFor(brightness)`
- _page_footer.dart: `AppLineTheme.subtleSecondaryFor(brightness)`
- landing_page.dart: `AppLineTheme.subtleSecondaryFor(brightness)`

To make them "slightly brighter for light theme," you'd adjust the light-theme hex values in `_AppLightColors` to be lighter (higher RGB values).
